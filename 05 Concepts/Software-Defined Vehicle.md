# Software-Defined Vehicle

## Summary

Software-Defined Vehicle (SDV) is a vehicle architecture and lifecycle concept in which software, data, updates, and connected services play a central role in vehicle functionality. In the EME WG SDV-AVS-AI meeting gap analysis, SDV is identified as a High priority missing concept and a core working group topic.

[VERIFY] A UNECE-specific formal definition has not been acquired yet. This note should not be treated as a regulatory definition.

## Background

The Eclipse Software Defined Vehicle initiative is an official industry initiative focused on open source software for SDVs. This provides a useful industry reference, but it is not a UNECE regulation or GRVA definition.

Within the UNECE Knowledge OS, SDV should be tracked as a cross-cutting concept because it can intersect with software updates, cybersecurity, automated driving systems, vehicle data, and AI-enabled vehicle functions.

## Purpose

This note provides a canonical place to track SDV as a concept when it appears in UNECE, GRVA, EME, or related regulatory discussions. It should link to formal regulatory material when such material is acquired.
