# Knowledge Evolution

Knowledge Evolution modernizes existing UNECE Knowledge OS notes to match the current Governance layer.

It is a structural migration skill. It does not research, acquire, merge new evidence, or review knowledge quality. It upgrades legacy structure while preserving existing meaning.

## Responsibility Boundary

| Skill | Responsibility |
|---|---|
| Acquire Knowledge | Creates or acquires knowledge from sources. |
| Knowledge Merge | Integrates verified new knowledge into canonical notes. |
| Knowledge Review | Evaluates quality, completeness, evidence, and gaps. |
| Knowledge Evolution | Modernizes existing knowledge structure to match Governance. |

## Core Outputs

- Migration plan before any edit
- Approved structural migration
- Evolution report after migration
- Dashboard/backlog updates when compliance improves

## Safety Principle

When uncertain, recommend. Do not modify.
